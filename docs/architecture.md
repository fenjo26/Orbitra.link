# Архитектура и Технологии

Проект Orbitra построен на современной, но легковесной связке технологий, что позволяет ему работать быстро даже на слабых серверах.

## Технологический стек

- **Backend:** PHP 8.x (чистый код без тяжелых фреймворков)
- **База Данных:** SQLite 3 (хранится в одном локальном файле)
- **Frontend:** React 18 + Vite (Single Page Application)
- **Стилизация:** Чистый CSS с использованием кастомных переменных (CSS Variables) для темизации
- **Иконки:** Lucide React
- ** HTTP Клиент:** Axios

## База Данных (SQLite)

Файл базы данных (`orbitra_db.sqlite`) создаётся автоматически при первом запуске через скрипт `config.php`.

### Основные таблицы
- `campaigns` — настройки рекламных кампаний.
- `streams` — потоки внутри кампаний (правила, фильтры, ротация).
- `offers` и `landings` — офферы и лендинги для распределения трафика.
- `clicks` — детализированная статистика по каждому переходу (IP, User-Agent, GEO).
- `conversions` — зарегистрированные лиды и продажи (с привязкой к клику).
- `traffic_sources` и `affiliate_networks` — источники трафика и партнерские сети.
- `domains` — паркованные домены для трекинга.

## Защита и Производительность

- **Анти-дребезг (Debounce):** В ядро встроена система защиты от двойных кликов, возникающих из-за агрессивной предварительной загрузки (prefetch/prerender) ссылок современными браузерами (Chrome, Safari).
- **Продвинутое управление доменами:** Встроена защита от поисковых ботов (перехват `/robots.txt` и отдача заголовка `X-Robots-Tag: noindex, nofollow`).

## Директории проекта

- `api.php` — единая точка входа для всех REST API запросов от React фронтенда.
- `index.php` — основной движок трекера: сюда поступают клики от пользователей, после чего вычисляется нужный поток и происходит редирект.
- `postback.php` — скрипт для приёма постбеков от партнёрских сетей (S2S интеграция).
- `frontend/` — исходный код React-приложения панели администратора.
