# Основные Функции Orbitra v0.9.3.1

Полный обзор функциональности трекера Orbitra — современного аналога Keitaro Tracker.

## 🎯 Управление кампаниями

### CampaignEditor — полноэкранный редактор кампаний

Редактор кампаний (`CampaignEditor.jsx`, 120KB) спроектирован для максимального удобства и вдохновлён интерфейсом Keitaro.

#### Вкладка "Основные"
- **Название кампании** — произвольное имя
- **Алиас** — авто-генерация 6 символов (keitaro-style)
- **URL кампании** — автоматически копируется в буфер
- **Домен** — выбор из добавленных доменов
- **Группа кампаний** — для организации
- **Источник трафика** — привязка к источнику
- **Код для лендингов** — JavaScript код для вставки на сайты

#### Вкладка "Финансы"
**6 моделей вознаграждения:**
- **CPC** (Cost Per Click) — оплата за клик
- **CPuC** (Cost Per Unique Click) — оплата за уникальный клик
- **CPM** (Cost Per Mille) — оплата за 1000 показов
- **CPA** (Cost Per Action) — оплата за действие (lead)
- **CPS** (Cost Per Sale) — оплата за продажу
- **RevShare** — процент от дохода партнёрки

**Настройки уникальности:**
- **Метод**: IP, IP+UA, Cookies
- **Срок уникальности**: 1-168 часов

**Ротация потоков:**
- **По весу** — пропорциональное распределение трафика
- **По позиции** — последовательное распределение

#### Вкладка "Параметры"
**30+ параметров** для передачи в оффер/лендинг:
- `keyword` — ключевые слова
- `cost` — стоимость клика
- `currency` — валюта
- `external_id` — внешний ID
- `creative_id` — ID креатива
- `ad_campaign_id` — ID рекламной кампании
- `source` — источник трафика
- `sub_id_1` ... `sub_id_30` — кастомные sub ID

#### Вкладка "Интеграции"
**5 предустановленных интеграций:**
- 📘 **Facebook** — автоматические параметры (fbclid, adset_id, campaign_id)
- 🎵 **TikTok** — параметры (ttclid, campaign_id, adgroup_id)
- 🔍 **Яндекс** — параметры (yclid, campaign_id, ad_id)
- 🔎 **Google Ads** — параметры (gclid, campaign_id, adgroup_id, creative)
- 💬 **VK Ads** — параметры (vk_click_id, vk_source_id)

#### Вкладка "S2S Postbacks"
- **URL для постбеков** — шаблон URL для отправки в партнерскую сеть
- **Метод** — GET или POST
- **Статусы** — выбор статусов для отправки (lead, sale, rejected, etc.)

#### Вкладка "Заметки"
- Текстовое поле для заметок к кампании

#### Дополнительные действия (кнопка "⋮")
- **Полный отчет (CSV)** — экспорт статистики в CSV
- **Лог конверсий** — просмотр конверсий по кампании
- **Симуляция трафика** — тестирование потоков
- **Обновить расходы** — ручное обновление стоимостей
- **Очистить статистику** — сброс статистики кампании

### Потоки (Streams)

**3 типа потоков:**
1. **Intercept** — перехват (высший приоритет, для особых случаев)
2. **Regular** — обычный поток (основной трафик)
3. **Fallback** — замыкающий (резервный поток)

**Фильтры потоков:**
- **Страна** — выбор стран (включая профили: EU, CIS, Americas)
- **Устройство** — desktop, mobile, tablet
- **ОС** — Windows, macOS, Linux, Android, iOS
- **Браузер** — Chrome, Safari, Firefox, Edge, Opera
- **ISP** — провайдер (регулярка)
- **IP** — белый/черный список
- **Keyword** — ключевые слова
- **Referer** — реферер (регулярка)
- **Время** — часы работы
- **День недели** — дни недели

**Схемы потоков:**
- **URL** — редирект на внешний URL
- **Offer** — редирект на оффер
- **Landing** — показ локального лендинга
- **Action** — выполнение действия (404, 200 OK, etc.)

## 🔗 Click API с токенами

### Что такое Click API?

Click API — это механизм для работы с интеграционными скриптами, когда вместо `campaign_id` используется уникальный токен кампании.

### Преимущества токенов
- **Совместимость с Keitaro** — существующие скрипты продолжат работать
- **Безопасность** — токен сложно угадать, ID легко перебрать
- **Гибкость** — можно менять кампании без изменения скриптов

### Как это работает

**Традиционный подход:**
```
https://tracker.com/click.php?campaign_id=123
```

**С токеном:**
```
https://tracker.com/click.php?token=abc123def456...
```

### Генерация токена

- **Длина**: 32 символа (hex)
- **Уникальность**: автоматическая генерация при создании кампании
- **Регенерация**: можно сгенерировать новый токен в редакторе

### Сохранение токенов при миграции

При импорте из Keitaro:
1. Токены из Keitaro автоматически переносятся в Orbitra
2. Сохраняется полная совместимость с существующими интеграциями
3. Интеграционные скрипты продолжат работать без изменений

### Где используется токен

- **Интеграционные скрипты** на лендингах
- **API запросы** от сторонних систем
- **Постбеки** от партнерских сетей
- **Вебхуки** от внешних сервисов

## 🔄 Миграция из Keitaro

### Подготовка дампа Keitaro

На сервере Keitaro выполните:

```bash
# 1. Подключитесь к Keitaro серверу
ssh root@YOUR_KEITARO_SERVER_IP

# 2. Создайте дамп настроек (без логов кликов)
bash -lc '
source /etc/keitaro/env/inventory.env

# Конфиг для подключения к БД
cat > /root/keitaro-mariadb.cnf <<EOF
[client]
user=$MARIADB_KEITARO_USER
password=$MARIADB_KEITARO_PASSWORD
host=127.0.0.1
port=3306
protocol=tcp
EOF
chmod 600 /root/keitaro-mariadb.cnf

# Создание дампа таблиц
TABLES="keitaro_affiliate_networks keitaro_groups keitaro_offers keitaro_domains keitaro_campaigns keitaro_campaign_postbacks keitaro_landings keitaro_streams keitaro_stream_filters keitaro_stream_offer_associations keitaro_stream_landing_associations keitaro_traffic_sources keitaro_ref_sources"

mysqldump --defaults-extra-file=/root/keitaro-mariadb.cnf \
  --single-transaction --quick --skip-lock-tables \
  "$MARIADB_KEITARO_DATABASE" $TABLES \
  | gzip > /root/keitaro_orbitra_full.sql.gz

ls -lah /root/keitaro_orbitra_full.sql.gz
'

# 3. Скачайте файл к себе
scp root@YOUR_KEITARO_SERVER_IP:/root/keitaro_orbitra_full.sql.gz .
```

### Импорт в Orbitra

1. Откройте **Миграции** в меню админки
2. Следуйте инструкции в блоке "Как создать бекап Keitaro"
3. Загрузите файл `keitaro_orbitra_full.sql.gz`
4. Выберите что импортировать:
   - ☑ Партнёрки (affiliate networks)
   - ☑ Офферы
   - ☑ Домены
   - ☑ Кампании
   - ☑ Потоки (streams)
   - ☑ Постбеки кампаний
   - ☑ Лендинги
   - ☑ Источники трафика
5. Нажмите **"Показать предпросмотр"** для проверки
6. Нажмите **"Импортировать в Orbitra"** для реального импорта

### Что импортируется

| Сущность Keitaro | Таблица Orbitra | Особенности |
|------------------|-----------------|-------------|
| `keitaro_affiliate_networks` | `affiliate_networks` | Сохраняются postback URL |
| `keitaro_offers` | `offers` | Сохраняются URL, payout, geo |
| `keitaro_domains` | `domains` | Сохраняется парковка, HTTPS |
| `keitaro_campaigns` | `campaigns` | **Сохраняются токены!** |
| `keitaro_streams` | `streams` | Сохраняются фильтры, схемы |
| `keitaro_landings` | `landings` | Сохраняются названия |
| `keitaro_traffic_sources` | `traffic_sources` | Сохраняются параметры |
| `keitaro_ref_sources` | — | Используется в streams |

### Особенности миграции

- **Токены сохраняются** — ваши скрипты продолжат работать
- **Streams filters** — фильтры конвертируются в новый формат
- **Postbacks** — S2S постбеки переносятся автоматически
- **Preview mode** — предварительный просмотр перед импортом

## 🤖 Telegram Bot

### Возможности бота

**Команды:**
- `/stats` — статистика за сегодня
- `/stats 7d` — статистика за 7 дней (1d, 30d, yesterday)
- `/campaigns` — список активных кампаний
- `/campaign 123` — детальная статистика кампании
- `/top` — ТОП-5 кампаний по доходу 🥇🥈🥉
- `/conversions` — последние 10 конверсий
- `/notify on|off` — вкл/выкл уведомлений
- `/daily on|off` — вкл/выкл ежедневной сводки
- `/lang ru|en` — язык бота
- `/help` — справка

### Уведомления

**Мгновенные уведомления:**
- При каждой конверсии
- Кампания, статус, сумма, валюта
- Страна с флагом
- Время конверсии

**Ежедневная сводка:**
- Cliks, конверсии, Revenue, Cost, Profit
- ТОП-3 кампании по доходу
- Настраиваемое время отправки

### Настройка бота

1. **Создайте бота** через `@BotFather` в Telegram
2. **Получите токен** бота
3. **Откройте** Интеграции → Telegram Bot
4. **Вставьте токен** → Подключить
5. **Отправьте** `/start` боту
6. **Готово!**
